TfL Data Pipeline

Project Overview

This project implements a modern data stack pipeline that processes real-time Transport for London (TfL) API data. The pipeline extracts data about tube lines, bus routes, and service disruptions, transforms it using dbt, and loads it into BigQuery for analysis and visualization.

Architecture

The pipeline follows a modern ELT (Extract, Load, Transform) architecture:

  • Extract: Python scripts fetch data from TfL's API endpoints
  • Load: Raw data is loaded directly into BigQuery staging tables
  • Transform: dbt models clean, transform, and model the data for analysis

Technical Implementation

Key Components:

  • Data Source: TfL's Open Data API providing real-time transport information
  • Data Warehouse: Google BigQuery for scalable data storage and analytics
  • Transformation: dbt for data modeling, testing, and documentation
  • Orchestration: Python scripts for API interaction and data ingestion

Data Models

The dbt transformation layer includes several key models:

  • Staging Models: Raw data from TfL API
  • Intermediate Models: Cleaned and standardized data
  • Mart Models: Business-ready datasets for analysis

Future Enhancements

  • Integration with real-time visualization tools
  • Extension to additional TfL services and API endpoints
  • Addition of Apache Airflow for orchestration and automation.
